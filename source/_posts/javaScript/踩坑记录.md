# Number的整数和安全整数

```typescript
// 最大的整数
console.log(Number.MAX_VALUE) // 1.7976931348623157e+308
(Math.pow(2, 53) - 1) * Math.pow(2, 971) // 1.7976931348623157e+308
// 最大安全值
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(2 ** 53 -1)
```



# WeakMap 和Map的区别

> node  --expose-gc index.js
> \--expose-gc 参数表示允许手动执行垃圾回收机制

map的堆内存使用情况

```typescript
function usedSize() {
  // 获取堆内存使用情况
  const used = process.memoryUsage().heapUsed;
  return Math.round((used / 1024 / 1024) * 100) / 100 + "M";
}

global.gc();
console.log(usedSize()); // 3.22M

const map = new Map();

let b = new Array(5 * 1024 * 1024);
map.set(b, 1);

b = null
global.gc()
// 此时的Array 无法被内存回收
console.log(usedSize()); // 43.28M

```

weakMap 内存使用情况

```javascript
function usedSize() {
  // 获取堆内存使用情况
  const used = process.memoryUsage().heapUsed;
  return Math.round((used / 1024 / 1024) * 100) / 100 + "M";
}

global.gc();
console.log(usedSize()); // 3.22M

const map = new WeakMap();

let b = new Array(5 * 1024 * 1024);
map.set(b, 1);

b = null
global.gc()
// 此时的Array 无法被内存回收
console.log(usedSize()); // 3.28M
```

可以看出 WeakMap是对引用类型的弱引用 不会限制引用类型被gc 而Map 会造成引用类型无法 gc 从而造成了内存泄漏。

> WeakMap的键是弱引用对象(包括null)
> WeakMap的弱引用只是键名 不是键值
> WeakMap 的 key 不可被枚举
